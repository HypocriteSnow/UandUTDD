# Unity 关卡编辑器 Workspace/Whitebox 成品化设计

> 本文档是当前有效设计基线。后续涉及地块、笔刷、出生点、目标点、传送点、颜色、覆盖规则、导出规则的开发，以本文为准。
>
> 本轮只处理地形与语义点的成品化重构。`路径` 保持独立模块，不并入地块类型体系，后续单独设计。

## 1. 产品目标

Unity 内的关卡编辑器应成为真正的关卡生产入口，而不是旧 `LevelConfig` 的附属查看器。

本轮目标是把 Unity 白模编辑流程调整为与网页编辑器一致的地块编辑模型：

1. 打开关卡编辑器。
2. 新建或打开 `Workspace`。
3. 输入 `Workspace` 名、导出名和基础地图参数。
4. 生成白模。
5. 使用统一笔刷直接编辑地图地块与语义点。
6. 保存 `Workspace`。
7. 导出 `<导出名>_LevelConfig`。

统一流程为：

`打开/新建 Workspace -> 输入参数 -> 生成白模 -> 用统一笔刷编辑地块与语义点 -> 保存 Workspace -> 导出 <导出名>_LevelConfig`

## 2. 本轮范围

### 2.1 在范围内

- `Workspace` 持久化、打开、保存、导出
- 基础地图参数编辑
- 白模生成与刷新
- 统一地块类型模型
- 统一笔刷类型选择区
- 出生点、目标点、传送点的显式编号
- 地块颜色表达
- 后刷覆盖前刷
- 去除 legacy fallback
- 校验与导出规则同步调整

### 2.2 不在范围内

- 路径系统详细交互重做
- 路径按钮并入地块类型区
- 路径自动生成策略优化
- 波次路径可视化重设计

本轮只要求路径继续作为独立模块存在，并且只引用地图中显式存在的出生点和目标点。

## 3. 核心原则

### 3.1 数据真源

- 编辑期真源是 `LevelEditorWorkspace`
- 白模是 `Workspace` 的可视化和交互载体
- `LevelConfig` 只是在导出时生成的运行时产物

### 3.2 单格单主类型

每个格子同一时刻只有一种主类型。

不采用“地形层 + 语义层”并存模型。

### 3.3 显式语义

出生点、目标点、传送点必须是地图中的显式对象。

不允许依赖旧的默认坐标、隐式入口、隐式终点、legacy fallback。

### 3.4 覆盖优先

地图编辑采用覆盖式规则。

后刷的类型覆盖前刷的类型，并移除旧类型附带的语义。

## 4. 工作区与命名规则

### 4.1 双命名字段

编辑器顶部保留两个字段：

- `Workspace 名称`
- `导出名称`

规则如下：

- 新建 `Workspace` 时，`导出名称` 默认等于 `Workspace 名称`
- 用户可单独修改 `导出名称`
- 提供“同步命名”按钮，把 `导出名称` 重置为 `Workspace 名称`
- `Workspace` 默认命名可使用英文，例如 `NewLevel`
- 导出资源默认命名仍为 `<导出名>_LevelConfig`

### 4.2 Workspace 资源化保存

`Workspace` 必须作为单独的编辑期资源保存。

建议目录：

- `Assets/Editor/LevelEditor/Workspaces/`

规则如下：

- 新建 `Workspace` 时创建一个 `LevelEditorWorkspaceAsset`
- 保存时写回当前 `Workspace`
- 打开时恢复基础参数、地图、语义点、波次、导出设置
- 导出 `LevelConfig` 时不覆盖 `Workspace` 资源

## 5. 地图数据模型

### 5.1 格子主类型

每个格子只有一个主类型。

主类型集合为：

- `禁用`
- `地面`
- `高台`
- `坑洞`
- `出生点`
- `目标点`
- `传送入口`
- `传送出口`

规则如下：

- 新建白模后，全图默认主类型为 `禁用`
- 不再默认整图为 `地面`
- `路径` 不是格子主类型
- `路径` 不参与本轮地块模型重构

### 5.2 显式编号

地图中的特殊点必须自动分配稳定编号。

规则如下：

- 第一个出生点为 `R1`
- 第二个出生点为 `R2`
- 第一个目标点为 `B1`
- 第二个目标点为 `B2`
- 第一个传送门对为 `IN1 / OUT1`
- 第二个传送门对为 `IN2 / OUT2`

编号分配策略：

- 采用“当前最大编号 + 1”
- 删除后不做全图重排
- 地图显示、波次引用、导出结果都使用同一编号

### 5.3 覆盖规则

地图编辑采用覆盖式规则。

规则如下：

- 当前格子是 `出生点`，再刷 `地面`，结果变为 `地面`
- 当前格子是 `目标点`，再刷 `高台`，结果变为 `高台`
- 当前格子是 `传送入口`，再刷 `禁用`，结果变为 `禁用`
- 普通地形也可被 `出生点`、`目标点`、`传送点` 覆盖
- 后刷的主类型始终替换前一个主类型

语义变化规则：

- 覆盖普通地形时，不保留旧语义
- 覆盖特殊点时，旧编号对象被移除
- 被移除编号如果仍被波次引用，进入校验错误，不自动修复

## 6. 白模显示规则

### 6.1 颜色表达

所有主类型都必须有可区分的颜色。

包括：

- `禁用`
- `地面`
- `高台`
- `坑洞`
- `出生点`
- `目标点`
- `传送入口`
- `传送出口`

规则如下：

- 颜色分类以网页编辑器为准
- Unity 不重新定义另一套分类含义
- 地块底色必须表达主类型
- 用户只看白模本身，也应能识别地图类型

### 6.2 特殊点辅助标记

允许保留当前特殊点的辅助标记。

规则如下：

- `出生点` 可保留球体/标记物
- `目标点` 可保留方框/标记物
- `传送入口/出口` 可保留辅助标记
- 但辅助标记不是唯一识别方式
- 对应格子的地面颜色也必须同步变化

### 6.3 编号显示

白模格子需要直接显示核心语义编号。

包括：

- `R1 / R2 / ...`
- `B1 / B2 / ...`
- `IN1 / OUT1 / ...`

## 7. 编辑器 UI 规则

### 7.1 笔刷工具结构

“笔刷工具”改为统一的地块类型选择区。

规则如下：

- `禁用`、`地面`、`高台`、`坑洞`
- `出生点`、`目标点`
- `传送入口`、`传送出口`

以上类型全部同级显示在一个按钮区中。

不再保留以下分裂结构：

- 上层 tab 选择 `地形 / 出生点 / 目标点 / 传送入口 / 传送出口`
- 下层按钮再选择 `地面 / 高台 / 禁用 / 坑洞`

新的交互模型要求：

- 用户先在统一按钮区选择一个主类型
- 再直接刷到地图上

### 7.2 UI 排版

笔刷区域的排版采用统一按钮矩阵样式。

要求：

- 风格参考当前下方 `地面 / 高台 / 禁用 / 坑洞` 这种规整布局
- 避免“上方 tab + 下方按钮”分裂布局
- 文案使用简体中文
- `Workspace`、`LevelConfig`、`ID` 可保留英文

### 7.3 路径位置

`路径` 继续作为独立模块存在。

规则如下：

- 不并入本轮统一地块类型按钮区
- 不作为格子主类型
- 仍通过独立面板和独立白模交互编辑

## 8. 波次与路径的当前边界

虽然本轮不重做路径系统，但波次和地图语义点的关系必须先固定。

规则如下：

- 波次中的 `spawnId` 必须来自地图中实际存在的 `R1 / R2 / ...`
- 波次中的 `targetId` 必须来自地图中实际存在的 `B1 / B2 / ...`
- 波次面板中显示的编号必须与地图上一致
- 如果引用目标已被覆盖删除，不能自动 fallback
- 这类情况必须在校验阶段明确报错

## 9. 导出与校验规则

### 9.1 导出规则

导出必须完全基于地图中的显式语义。

规则如下：

- 出生点集合来自地图中的全部 `R*`
- 目标点集合来自地图中的全部 `B*`
- 传送门集合来自地图中的全部有效 `IN* / OUT*`
- 不允许导出阶段补默认出生点
- 不允许导出阶段补默认目标点
- 不允许再使用旧隐式坐标

### 9.2 校验规则

至少需要校验：

- 地图尺寸合法
- 导出名合法
- 地图中存在可用出生点
- 地图中存在可用目标点
- 波次引用的 `spawnId` 存在
- 波次引用的 `targetId` 存在
- 传送门配对完整
- 导出后的运行时结构完整

校验失败时：

- 阻止导出
- 在编辑器中列出明确问题
- 尽量定位到具体波次或具体对象

## 10. 对当前实现的修正

当前仓库中的实现已经具备以下基础：

- `Workspace -> Whitebox -> Export LevelConfig` 的主架构方向
- 白模生成与刷新
- 基础导出
- 基础波次与路径数据结构

但以下假设已经被本设计明确废止：

- 默认整图为 `地面`
- `出生点/目标点` 作为独立上层工具，而不是统一地块类型
- 出生点与目标点只靠小标记区分，不改地块颜色
- 地块和语义点不能互相覆盖
- 导出阶段继续使用隐式 fallback

## 11. 验收标准

当以下条件全部满足时，可认为本轮设计达标：

1. 打开编辑器，新建地图后，白模默认全图为 `禁用`
2. 用户可直接刷出 `地面`、`高台`、`坑洞`
3. 用户可直接刷出 `出生点`、`目标点`、`传送入口`、`传送出口`
4. 地图上的特殊点自动生成并显示 `R1 / B1 / IN1 / OUT1` 等编号
5. 特殊点既有辅助标记，也有对应地块底色变化
6. 不同主类型可通过颜色直接区分
7. `出生点/目标点/传送点` 与普通地形可以互相覆盖
8. 波次只能引用地图上实际存在的 `R* / B*`
9. 导出不再依赖 legacy fallback
10. 路径模块继续独立存在，不被本轮重构破坏

## 12. 结论

本轮成品化方向不是继续补丁式扩展旧编辑器，而是统一地图主类型模型，统一笔刷交互，统一颜色表达，统一编号语义，统一导出规则。

当前有效流程为：

`打开/新建 Workspace -> 输入参数 -> 生成白模 -> 选择统一地块类型 -> 刷写地图 -> 保存 Workspace -> 导出 <导出名>_LevelConfig`
